# 복약 트래커

복용 간격과 하루 최대치를 기준으로 **다음 복용 가능 시각**을 계산해 주는 로컬 전용 PWA.
서버 없이 모든 데이터는 기기의 브라우저(localStorage)에만 저장됩니다.

## 두 가지 화면 모드

첫 실행 시 온보딩에서 선택하고, 설정에서 언제든 변경할 수 있습니다.

### 간격 트래커 모드
- 약마다 최소 복용 간격(시간)과 1일 최대 개수 등록 (수정·삭제·추가 가능)
- 복용 기록 시 다음 복용 가능 시각까지 남은 시간을 **원형 카운트다운 링**으로 표시
- 오늘 복용량 / 일일 최대치 배지, 최대치 도달·초과 시 경고 배너
- 날짜별 복용 이력 목록
- 기본 프리셋: 타이레놀 500mg (간격 4시간, 1일 최대 8정), 이지엔6프로 (간격 4시간, 1일 최대 6캡슐)

### 심플 체크 모드
- 아침 · 점심 · 저녁 시간대별 큰 체크 버튼 — 탭 한 번으로 기록
- 마지막으로 먹은 시각을 큰 글씨로 표시
- 체크한 시간대에만 작은 취소 버튼 노출 (오입력 복구)
- 큰 글씨 · 큰 버튼 · 최소 요소 (고령자 접근성 우선)

## 기술 구성
- Vanilla JS 단일 페이지, 프레임워크·빌드 도구 없음
- `localStorage` 저장 (서버 전송 없음), 날짜가 바뀌면 "오늘 복용량"만 자동 리셋되고 이력은 보존
- PWA: `manifest.json` + 서비스워커(`sw.js`) — 홈 화면에 추가해 앱처럼 설치, 오프라인 동작
- 모바일 우선 반응형, 다크모드(`prefers-color-scheme`) 대응
- 상대 경로만 사용 → GitHub Pages 프로젝트 경로(`/med-tracker/`)에서 그대로 동작

## GitHub Pages 배포
1. 저장소 **Settings → Pages → Source**를 **GitHub Actions**로 설정
2. `main` 브랜치에 푸시하면 `.github/workflows/deploy.yml`이 자동 배포
3. 배포 주소: `https://<계정>.github.io/med-tracker/`

## 참고
- 푸시 알림은 현재 범위에서 제외 (추후 확장 예정)
- 이 앱은 의학적 판단을 하지 않습니다. 사용자가 등록한 간격·최대치를 기준으로 계산만 합니다.
